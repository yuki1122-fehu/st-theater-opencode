# 小剧场生成器 / Theater Generator

独立生成小剧场/番外内容的 SillyTavern 扩展插件。在插件面板中调用API生成HTML小剧场，不影响正文剧情。

## 功能

- **独立生成** — 小剧场在插件面板内独立生成，完全不影响正文
- **预设选择** — 从酒馆已导入的 Chat Completion 预设中选择，按条目开关，独立于正文预设
- **世界书切换** — 下拉选择世界书或直接导入JSON文件，支持按条目展开/开关/收起
- **流式传输** — 主API和独立API均支持SSE流式传输，实时显示生成内容
- **智能重试** — 遇到502/524/529自动重试，token过长时提前警告
- **指令模板** — 保存常用指令，支持txt文件批量导入导出（---分隔），折叠收纳
- **渲染规则** — 控制输出格式，记住上次选择
- **交互模式** — 可生成带点击/交互功能的小剧场
- **历史记录** — 保存生成结果，支持单条导出和批量导出
- **悬浮球** — 可选悬浮球快速打开面板，设置中开关
- **主题跟随** — 面板自动跟随酒馆主题配色
- **自定义CSS** — 自定义插件面板样式
- **安全更新** — 面板内更新扩展，不刷新页面不影响其他插件

## 安装

### 方法一：扩展管理器（推荐）

Extensions → Install Extension → 粘贴仓库地址

### 方法二：手动安装

将文件放入 `data/default-user/extensions/third-party/st-theater/`

## 面板结构

1. **生成** — 输入指令、流式输出、生成结果
2. **素材** — 生成预设、User人设、世界书、上下文条数
3. **规则** — 指令模板库、渲染规则模板
4. **历史** — 保存的小剧场、单条/批量导出
5. **美化** — 自定义CSS
6. **设置** — API配置、扩展更新

## 兼容性

- SillyTavern 1.12.0+
- 支持所有 API

## OpenCode 适配说明（本分支）

本分支（OpenCode 适配版）针对「酒馆能连上、但插件连不上」的问题做了修复：
OpenCode 等云端 OpenAI 兼容端点**不返回 CORS 头**，浏览器直连会被拦截。

修复方式：把「独立 API」模式的生成请求改为走酒馆自己的同域后端代理
`/api/backends/chat-completions/generate`（服务端转发，无 CORS 限制），
**不改动全局 `window.fetch`**，不会影响酒馆其他功能。

### v4.1.0 更新内容

- **获取模型列表 & 测试连接也走后端代理** — 修复了「获取模型列表」和「测试连接」仍走浏览器直连导致 CORS 失败的问题，现在与生成请求一致，均经酒馆后端代理转发。
- **API 格式可选** — 新增 API 格式下拉（自动检测 / OpenAI 兼容 / Anthropic），不再仅靠 URL 猜测，代理 URL 访问 Claude 模型也能正确解析流式响应。
- **最大输出 token 可调** — 新增「最大输出 token」设置项（默认 8192），不再硬编码，适配不同模型的上限。
- **智能重试** — 遇到 502/524/529 网关错误时自动重试（最多 2 次，递增延迟），而非直接报错。
- **续写截断优化** — 续写前情内容超过 8000 字时，在段落边界截断而非从句子中间断开。
- **SSE 流式解析修复** — Anthropic 格式的独立 API 请求现在正确传入 `isAnthropic` 标志，流式解析不再错误走 OpenAI 格式。

### OpenCode 配置步骤

1. 扩展管理器安装后，**硬刷新**网页（Ctrl+Shift+R）。
2. 打开插件面板 → 设置 → API 模式选「**独立 API**」。
3. 填写：
   - **API URL**：`https://你的-opencode 地址/zen/go/v1`（即 OpenAI 兼容 base URL，不用带 `/chat/completions`）
   - **API Key**：OpenCode 给的 key
  - **模型名称**：对应模型（点「获取模型列表」若因 CORS 失败，直接手填即可）
  - **API 格式**：一般选「自动检测」即可；若用代理 URL 访问 Claude 模型，请选「Anthropic」
  - **最大输出 token**：根据模型上限调整（默认 8192）
4. 保存后直接生成即可，请求会经酒馆后端代理转发到 OpenCode。

> 若想用酒馆主 API，API 模式选「酒馆主 API（实验）」即可，同样无需担心 CORS。
